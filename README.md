# Innovation Information Initiative (I³) Website

The official website for the Innovation Information Initiative, built with [Quarto](https://quarto.org/). This is a clean, modern static website ready for content migration from PubPub.

## Prerequisites

- [Quarto](https://quarto.org/) - Install by following the [official installation guide](https://quarto.org/docs/get-started/)

## Quick Start

### Preview Locally
```bash
quarto preview
```
This starts a local development server with live reload.

### Build the Website
```bash
quarto render
```
This generates static files in the `docs/` directory for deployment.

## Content Migration

See `MIGRATION_GUIDE.md` for detailed instructions on migrating content from PubPub.

### Adding Content

Content is managed through YAML files and individual pages:

- **News**: Add entries to `news.yml`
- **Events**: Add entries to `events.yml`
- **Publications**: Add entries to `publications.yml`
- **Essays**: Create individual `.qmd` files or use a YAML file
- **Datasets**: Update `datasets.qmd` with dataset information

### Images

Store images in the `/files/images/` directory:
- `/files/images/news/` - News images
- `/files/images/events/` - Event images
- `/files/images/journal/` - Journal/publication logos

## Configuration

### Site Settings

Edit `_quarto.yml` to configure:
- Site title and description
- Navigation menu
- Theme and styling
- Search functionality
- Social media integration

### Styling

- `styles.css` - Main stylesheet
- `custom.css` - Homepage custom styles
- Navbar colors defined in `_quarto.yml`

### Analytics

Add your analytics code to `/files/includes/_msclarity.qmd`

## Deployment

### GitHub Pages

1. Build the site: `quarto render`
2. Commit the `docs/` directory
3. Enable GitHub Pages in repository settings, pointing to the `docs/` folder
4. Update the site URL in `_quarto.yml` when deployed
